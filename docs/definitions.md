## Backpatch

A security or reliability patch applied to an older software line that is not receiving the same fix through normal upstream maintenance.

## Patch provider

An organization or participant that produces, validates, publishes, or contributes OSERA-compatible backpatches.

## Recipient

An enterprise, vendor, or other downstream consumer that evaluates and applies OSERA-compatible patches.

## Baseline tag

The tag identifying the original source state from which a backpatch line begins, using the `v<VERSION>+backpatch.baseline` scheme.

## Patch basis

The reason and source for a patch change. A patch basis can be an upstream commit backported to an older line, or a provider-developed fix where no upstream fix exists.

## Recipient evidence

Metadata and human-readable guidance that helps recipients understand what changed, why it changed, and what they should test before adopting the patch.
