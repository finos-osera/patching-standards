# Scope

The OSERA Patching Standards Working Group develops specifications, guidance, examples, and supporting evidence formats for producing, publishing, discovering, and consuming open source backpatches through FINOS OSERA.

The scope includes:

* naming, branching, tagging, and fork-management conventions for OSERA backpatch repositories;
* source-change provenance, including links to upstream fixes and identification of provider-developed fixes;
* release and compatibility requirements, including version metadata and bytecode compatibility for Java artifacts;
* vulnerability and advisory feed expectations, including OpenVEX and CycloneDX;
* provider evidence that helps recipients understand what changed, why it changed, and what application surface area should be tested;
* examples and acceptance checks that help patch providers and enterprise recipients implement the standards consistently.

The scope does not require OSERA to define a single provider build system, mandate a single public CI platform, or replace upstream project governance. Source code examples or tools included in this repository are non-normative unless a specification explicitly states otherwise.

Any changes of Scope are not retroactive.
